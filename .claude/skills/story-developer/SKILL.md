# 스토리 개발 스킬 (Story Developer)

## 개요

이 스킬은 **AI 영상 제작을 위한 스토리 분석 및 구조화**를 담당합니다. 사용자가 제공한 복잡한 드라마 스토리를 5개의 에피소드로 분해하고, 각 에피소드의 감정 호(emotion arc)와 시각적 요소를 정의하여 영상 프롬프트 생성의 토대를 마련합니다.

## 핵심 작업

### 1. 스토리 분석 (Story Breakdown)

**입력받은 전체 스토리를 3단계로 분석:**

| 단계 | 내용 | 산출물 |
|-----|------|--------|
| 1. 갈등 식별 | 주요 갈등 3~4가지 추출 | 갈등 목록 |
| 2. 감정 곡선 | 전체 스토리의 감정 변화 추적 | 감정 그래프 |
| 3. 전환점 | 스토리 전개의 핵심 터닝포인트 | 5개 에피소드의 분기점 |

**인어왕자 스토리 예시:**
- 갈등 1: 정략결혼 vs 자유로운 사랑
- 갈등 2: 인어 세계 vs 인간 세계
- 갈등 3: 기억 상실과 정체성 혼란
- 전환점: 인간여성의 재등장 → 기억 복구 → 인간 변신 결심

### 2. 에피소드 분해 (Episode Structuring)

**5부작 분배 원칙:**

```
EP1: 불운한 정혼 (설정 + 첫 만남)
     → 인어왕자의 불안, 인간여성 구조

EP2: 금지된 사랑 (감정 고조)
     → 사랑이 깊어짐, 위험한 관계

EP3: 강압과 도망 (갈등 폭발)
     → 다른 인어공주의 등장, 왕자 도망

EP4: 기억의 바다 (전환점)
     → 왕자 생명 위기, 기억 상실, 새로운 사랑

EP5: 깨어난 진실 (해결 + 결말)
     → 기억 회복, 진정한 선택, 인간 변신
```

**각 에피소드 분석 항목:**
- 시작 상태 (Starting situation)
- 핵심 갈등 (Central conflict)
- 클라이맥스 (Climax moment)
- 종료 상태 (Ending state)
- 감정 곡선 (Emotional arc)

### 3. 감정 곡선 설계 (Emotion Arc Design)

각 에피소드마다 **4단계 감정 진행:**

```
오프닝 (Opening) 
  ↓ 감정: 상황 인식 (0~15초)
  
클라이맥스 (Climax)
  ↓ 감정: 갈등 폭발 (15~30초)
  
반응 (Response)
  ↓ 감정: 감정 변화 (30~45초)
  
전환 (Transition)
  ↓ 감정: 다음 장면으로 연결 (45~60초)
```

**감정 표현 방식:**
- 색감: 따뜻함(오렌지/붉음) vs 차가움(파랑/청동)
- 음향: 클래식/서정적 vs 긴장감 있는 사운드
- 속도: 느린 전개 vs 빠른 교차

### 4. 시각 가이드 생성 (Visual Guidelines)

각 씬별 시각적 요소 정의:

```
배경:
  - 인어 궁전 (화려한 산호, 보석)
  - 인간 세계 (해변, 도시)
  - 깊은 바다 (어두운, 신비로움)

색감 팔레트:
  - EP1~2: 밝고 화려함
  - EP3~4: 어두워지고 불안정함
  - EP5: 밝아지며 희망

조명:
  - 수중: 푸른 빛, 섬세한 광선
  - 육지: 자연광, 따뜻한 톤
  - 클라이맥스: 극적인 역광/드라마틱 조명
```

### 마크다운 출력 (사용자용)

```markdown
# 인어왕자의 사랑과 선택 - 5부작 분석

## EP1: 불운한 정혼

**스토리 요약:** 인어왕자가 왕국의 정략결혼을 강요받으며 시작...

**감정 곡선:** 불안 → 설렘 → 충격

**주요 색감:** 깊은 파랑, 산호 핑크, 은빛

### 씬 1-1: 정혼식 준비
- 배경: 화려한 산호 궁전, 반짝이는 보석으로 장식
- 등장인물: 인어왕자 (불안한 표정), 정략혼인 인어공주 (기대에 찼음)
- 감정: 화려함 속의 불안감
- 음향: 클래식한 궁전음악에 불안한 현악기

...
```

## 작업 지침

### DO: 스토리 분석 시 권장

✅ 사용자의 스토리를 **그대로 존중**하되, 논리적 일관성 확인  
✅ 각 에피소드가 **독립적인 갈등과 해결**을 가지도록 구조화  
✅ 감정 곡선이 **5개 에피소드에서 변화**되도록 설계  
✅ 인어 세계와 인간 세계의 **시각적 구분** 명확히  
✅ 캐릭터의 **심리 상태 변화**를 명시  

### DON'T: 피해야 할 것

❌ 사용자의 스토리를 임의로 수정하지 마시오  
❌ 에피소드가 너무 비슷하거나 중복되지 않게 주의  
❌ 색감/음향 가이드를 추상적으로 남기지 말 것 (구체화 필요)  
❌ 에피소드 분량 불균형 (±10초 범위 내)  

## 재호출 시 행동

이전 스토리 분석이 존재하면:
- 파일 읽고 기존 구조 존중
- 사용자 피드백이 주어지면 해당 부분만 수정 (전체 재작성 금지)
- 수정 사유와 변경 부분을 명확히 기록

## 테스트 시나리오

**테스트 1: 기본 스토리 분석**
입력: "인어왕자가 정략결혼을 거부하고 인간여자와 사랑하는 막장 드라마"
기대: 5부작 에피소드 구조 + 감정 곡선 + 시각 가이드 완성

**테스트 2: 복잡한 다층 갈등**
입력: 사용자 제공 인어왕자 스토리 (정략결혼 + 기억상실 + 정체성 혼란)
기대: 모든 갈등이 5개 에피소드에 균등 분배, 각 에피소드마다 해결점 제시
