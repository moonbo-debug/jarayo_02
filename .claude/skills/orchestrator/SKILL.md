# AI 영상 제작 오케스트레이터 (AI Drama Orchestrator)

## 개요

**목표:** 사용자의 드라마 스토리를 입력받아, AI 영상 생성용 **5부작 프롬프트 세트(총 20개)**를 자동으로 생성하고 정리된 문서로 제공.

**입출력:**
- **입력:** 드라마 스토리 (자유로운 형식)
- **출력:** 
  1. 5부작 스토리 분석 (각 에피소드 핵심 가이드)
  2. 에피소드별 프롬프트 4개 × 5회 = 총 20개
  3. 캐릭터 레퍼런스 시트
  4. 제작 가이드 및 체크리스트

**실행 모드:** 에이전트 팀 (스토리 아키텍트 + 프롬프트 생성자 × 5 + 코디네이터)

---

## Phase 0: 컨텍스트 확인

**실행 시작 전 다음을 확인:**

```
_workspace/ 디렉토리 존재?
  ├─ YES: 기존 산출물이 있음 → 사용자 요청 유형 판별
  │        ├─ "다시 만들어줘" → 새 실행 (_workspace → _workspace_prev로 이동)
  │        ├─ "수정해줘" → 부분 재실행 (해당 에이전트만)
  │        └─ "추가로" → 기존 기반 확장
  └─ NO: 첫 실행 → 전체 신규 구성
```

## Phase 1: 입력 검증 & 스토리 확인

### 1-1. 사용자 입력 검증

사용자가 제공한 스토리에서 다음을 확인:

- [ ] 스토리 개요가 명확한가? (누가, 뭘 하는가?)
- [ ] 주요 갈등이 3개 이상 있는가?
- [ ] 시작과 종료가 명확한가?

**부족한 경우:** 사용자에게 확인 질문

```
예: "스토리에서 주인공의 최종 선택(인간이 되기)이 명확히 보이는데,
     인간 세계에 대한 구체적 배경(도시? 시골? 현대?)이 있으면 프롬프트 작성이 더 정교할 것 같습니다.
     구체적 배경 정보가 있나요?"
```

### 1-2. 처리 방식 결정

```
입력 스토리가 충분히 구체적?
  ├─ YES → Phase 2로 진행 (팀 생성)
  └─ NO → 사용자와 협의 후 진행
```

## Phase 2: 팀 생성 & 작업 할당

### 2-1. 팀 구성

```
팀 구성원:
  ├─ 스토리라이터 (story-architect)
  ├─ 프롬프트 크리에이터 EP1 (prompt-generator-ep1)
  ├─ 프롬프트 크리에이터 EP2 (prompt-generator-ep2)
  ├─ 프롬프트 크리에이터 EP3 (prompt-generator-ep3)
  ├─ 프롬프트 크리에이터 EP4 (prompt-generator-ep4)
  ├─ 프롬프트 크리에이터 EP5 (prompt-generator-ep5)
  └─ 코디네이터 (coordinator)
```

### 2-2. 작업 할당

**Task 1: 스토리 분석 (스토리라이터가 수행)**
```
Title: 전체 드라마 스토리 분석 및 5부작 분해
Description:
  - 사용자 스토리를 5개 에피소드로 분해
  - 각 에피소드의 핵심 갈등, 감정 곡선, 시각 가이드 생성
  - JSON 및 마크다운 포맷 산출물 생성
Expected Output: 
  - 5부작 분석 (JSON)
  - 각 에피소드별 가이드 (마크다운)
```

**Task 2~6: 프롬프트 생성 (EP별 프롬프트 크리에이터가 병렬 수행)**
```
Task 2: EP1 프롬프트 생성
  - 의존: Task 1 완료 (스토리 아키텍트의 출력)
  - 산출물: 4개 프롬프트

Task 3: EP2 프롬프트 생성
  - 의존: Task 1 완료
  - 산출물: 4개 프롬프트

... (동일 패턴)

Task 6: EP5 프롬프트 생성
  - 의존: Task 1 완료
  - 산출물: 4개 프롬프트
```

**Task 7: 통합 & 최종 문서 (코디네이터가 수행)**
```
Title: 전체 프롬프트 통합 및 최종 문서 작성
Description:
  - Task 1~6의 모든 산출물 수집
  - 캐릭터/배경 일관성 검증
  - 최종 마스터 문서 작성
Expected Output:
  - 마스터 프롬프트 가이드 (모든 에피소드 통합)
  - 캐릭터 레퍼런스 시트
  - 제작 체크리스트
```

## Phase 3: 팀 실행

### 3-1. 스토리라이터 시작

```
SendMessage(스토리라이터):
"다음 스토리를 분석해주세요:

[사용자 스토리]

산출물:
1. 5부작 에피소드 분해 (JSON)
2. 각 에피소드별 시각 가이드 및 감정 곡선

완료 후 각 프롬프트 크리에이터에게 EP별 가이드를 전달해주세요."
```

### 3-2. 프롬프트 크리에이터 병렬 실행

스토리라이터 완료 후 각 EP별 크리에이터에게:

```
SendMessage(프롬프트-크리에이터-EP1):
"[EP1 스토리 가이드]

이 가이드를 기반으로 4개의 15초 프롬프트를 작성해주세요.
- 프롬프트 1: 0~15초
- 프롬프트 2: 15~30초
- 프롬프트 3: 30~45초
- 프롬프트 4: 45~60초

각 프롬프트는 마크다운 포맷으로 작성하고,
배경/등장인물/감정/색감을 명시해주세요."
```

*(동일 메시지를 EP2~5 크리에이터에게 전송)*

### 3-3. 코디네이터 최종 통합

모든 프롬프트 수집 후:

```
SendMessage(코디네이터):
"다음 산출물들을 수집했습니다:
- 스토리 분석: [링크]
- EP1~5 프롬프트: [각 링크]

다음을 수행해주세요:
1. 전체 프롬프트 통합 문서 작성
2. 캐릭터 일관성 검증 (외형/성격 변화)
3. 배경 일관성 검증 (색감, 분위기)
4. 최종 제작 가이드 작성"
```

## Phase 6: 에러 핸들링

### 6-1. 에러 유형별 대응

| 에러 | 원인 | 대응 |
|------|------|------|
| 프롬프트 누락 | 팀원 작업 미완료 | 해당 팀원에 재작업 지시 |
| 캐릭터 불일치 | 프롬프트 간 설정 다름 | 코디네이터가 자동 수정 |
| 스토리 오류 | 스토리 분석 오류 | 스토리라이터에 재검토 요청 |
| 시간 초과 | 팀원 응답 지연 | 받은 산출물만으로 최종 문서 작성 |

### 6-2. 재시도 정책

실패한 작업:
1. 첫 재시도: 동일 팀원에게 재작업 지시
2. 두 번째 실패: 다른 팀원 또는 오케스트레이터가 직접 처리

## 테스트 시나리오

### 테스트 1: 기본 흐름 (신규 요청)

**입력:**
"인어왕자가 정략결혼을 거부하고 인간여자와 사랑하는 막장 스토리로 5부작 영상 프롬프트 만들어줘"

**기대 출력:**
1. 5부작 스토리 분석
2. 총 20개 프롬프트 (EP별 4개)
3. 캐릭터 레퍼런스
4. 제작 가이드

**검증:**
- [ ] 5부작이 균등 분배되었나? (각 ~60초)
- [ ] 20개 프롬프트가 모두 생성되었나?
- [ ] 캐릭터가 일관되게 표현되었나?
- [ ] 감정 곡선이 연결되는가?

### 테스트 2: 부분 수정 (기존 결과 있을 때)

**입력:**
"EP3의 톤을 더 극적으로 해줘"

**기대:**
- 기존 _workspace/ 인식
- EP3 프롬프트만 재작성
- 나머지는 유지
- 최종 문서 업데이트

## 노트

- **컨텍스트 효율성:** 이 오케스트레이터 스킬 자체는 `story-developer`와 `video-prompt-generator` 스킬에 위임. 오케스트레이터는 팀 조율만 담당.
- **팀 크기:** 스토리라이터 1 + 프롬프트 크리에이터 5 + 코디네이터 1 = 7명. 에피소드별 병렬 처리로 빠른 완료.
- **재사용성:** 이 오케스트레이터는 어떤 드라마 스토리든 적용 가능. 5부작 구조만 고정, 나머지는 유연.
